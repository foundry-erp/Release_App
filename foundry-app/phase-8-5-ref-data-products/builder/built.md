## How To Reach Each Deliverable

### GET /api/products
Deployed to Vercel via `vercel --prod` from foundry-app/backend/. Route: `api/products/index.js`. Accessible at `https://foundry-app-rouge.vercel.app/api/products`. Requires `Authorization: Bearer <JWT>` header.

### PATCH /api/products/update
Deployed to Vercel via same backend deploy. Route: `api/products/update.js`. Accessible at `https://foundry-app-rouge.vercel.app/api/products/update`. Body: `{ id: "<uuid>", description: "<string>" }`.

### POST /api/sync — update_product_description
Handled in existing `api/sync/index.js`. Send action `{ type: "update_product_description", payload: { moduleSlug: "quality-inspector", payload: { productId: "<uuid>", description: "<string>" } }, local_id: "<id>" }` in the actions array.

### quality-inspector App.jsx v1.2.0
Built bundle at `modules/quality-inspector/dist/bundle.js`. Upload to CDN by running `node upload-to-supabase.js` from `foundry-app/phase-5-module-cdn/`. Then run the SQL activation steps in Supabase dashboard.

### CDN Upload + SQL Activation
1. Run `node upload-to-supabase.js` from `foundry-app/phase-5-module-cdn/` — uploads quality-inspector/1.2.0/bundle.js and index.html to Supabase Storage bucket `module-bundles`.
2. In Supabase SQL editor run:
   ```sql
   UPDATE module_versions SET is_active = false
     WHERE module_id = (SELECT id FROM modules WHERE slug = 'quality-inspector');
   INSERT INTO module_versions (module_id, version, cdn_url, index_url, checksum, is_active)
     VALUES (
       (SELECT id FROM modules WHERE slug = 'quality-inspector'),
       '1.2.0',
       'https://gbjmxskxkqyfvqifvelg.supabase.co/storage/v1/object/public/module-bundles/quality-inspector/1.2.0/bundle.js',
       'https://gbjmxskxkqyfvqifvelg.supabase.co/storage/v1/object/public/module-bundles/quality-inspector/1.2.0/index.html',
       '',
       true
     );
   ```
