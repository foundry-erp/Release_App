## Issues Encountered and Resolved

### [BLOCKING — RESOLVED] upload-to-supabase.js dependency resolution
The validator flagged that `require('@supabase/supabase-js')` in the upload script would fail because `backend/node_modules/` is not on the ancestor path of `phase-5-module-cdn/`. Fixed by creating `phase-5-module-cdn/package.json` with `@supabase/supabase-js: ^2.39.0` as a local dependency. The user must run `npm install` inside `phase-5-module-cdn/` before running the upload script.

### [WARNING — ACKNOWLEDGED] PostgREST !inner join syntax
The rewritten `backend/api/modules/index.js` uses `!inner` in the Supabase `.select()` call to filter out soft-deleted modules. This requires PostgREST v10+. An inline code comment in the file explains the fallback: remove `!inner` and post-filter in JS if the query fails on older PostgREST.

### [NOTE — RESOLVED] Pre-existing schema mismatch
The old `backend/api/modules/index.js` queried `index_url` and `bundle_checksum` from the `modules` table, but those columns do not exist in `schema.sql`. The Phase 5 rewrite replaces this query entirely with the `user_module_permissions → module_versions → modules` join, so these missing columns are no longer referenced.

## What the User Still Needs to Do Manually

1. **Install phase-5-module-cdn dependencies:**
   ```bash
   cd phase-5-module-cdn
   npm install
   cd ..
   ```

2. **Create Supabase Storage bucket:**
   - Supabase Dashboard → Storage → New bucket
   - Name: `module-bundles`
   - Public: ON
   - CORS: allow all origins (*)

3. **Upload bundles to Supabase Storage:**
   ```bash
   SUPABASE_URL=https://<ref>.supabase.co \
   SUPABASE_SERVICE_ROLE_KEY=<service_role_key> \
   node phase-5-module-cdn/upload-to-supabase.js
   ```
   Windows PowerShell:
   ```powershell
   $env:SUPABASE_URL="https://<ref>.supabase.co"
   $env:SUPABASE_SERVICE_ROLE_KEY="<service_role_key>"
   node phase-5-module-cdn/upload-to-supabase.js
   ```

4. **Run SQL migration** (`backend/sql/migrate_phase5.sql`) in Supabase SQL editor after replacing:
   - `<SUPABASE_PROJECT_REF>` → your project ref (from Supabase → Settings → API)
   - `<QI_CHECKSUM>` → `2a4a07c1536ca098800b5650dafa91fc25f9d10ea77779071fb809ee624ed336`
   - `<IC_CHECKSUM>` → `909943b82e9a6c62a391d5de1b7a6066e6b130b65a8717c1bcfae7365bb0c70e`
   - `<YOUR_FIREBASE_UID>` → Firebase UID of the test user (optional — only needed for pre-seeding permissions without using app login flow)

5. **Verification tests:**
   - GET /api/modules with a valid JWT → array with cdn_url + checksum per module
   - Open Supabase Storage URL in browser → module UI renders
   - Manual checksum: download bundle.js → sha256 → matches DB value
   - CORS test: fetch() from browser DevTools → no CORS error
