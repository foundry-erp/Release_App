# Phase 11 — Desktop Platform Support (macOS + Windows)
PHASE_ID: phase-11-desktop-platform
PLANNER_DOC_VERSION: 6.1.0
DEPENDS_ON: [phase-10-permission-revocation, phase-9-android-playstore]
PROVIDES_TO: [none — additive feature]

## What This Phase Builds

Makes the Flutter app compile and run correctly on macOS and Windows desktops,
in addition to the existing Android and iOS mobile targets.

The module system (WebView + React + offline-core) works identically on all platforms.
Camera and barcode scanner are mobile-only capabilities — on desktop they are replaced
with a file picker (camera) and hidden entirely (barcode).

## Key Findings From Reading Existing Code

- `module_webview.dart` imports `webview_flutter_android` unconditionally at the top —
  this causes a compilation failure on Windows/macOS builds
- `shell_bridge.dart` calls `ImagePicker(camera)` and `BarcodeScannerScreen` with no
  platform guards — both crash on desktop
- macOS entitlements missing: network client, outgoing connections, file read access
- `webview_flutter_wkwebview` already in pubspec — supports macOS (WKWebView) already
- Windows needs `webview_flutter_windows` added (WebView2, built into Windows 10/11)
- Firebase needs cross-platform config — currently only `google-services.json` (Android)
  and `GoogleService-Info.plist` (iOS) exist; macOS and Windows need their own config

## Requirements Covered

- REQ-11.1: App compiles and runs on macOS without errors
- REQ-11.2: App compiles and runs on Windows without errors
- REQ-11.3: WebView loads React modules correctly on macOS and Windows
- REQ-11.4: Camera bridge method uses file picker on desktop (no crash)
- REQ-11.5: Barcode bridge method returns graceful "not supported" on desktop
- REQ-11.6: macOS entitlements allow network access and local HTTP server
- REQ-11.7: Firebase initializes correctly on macOS and Windows

## Deliverables

- [ ] `pubspec.yaml`: add `webview_flutter_windows: ^0.2.0`, `file_picker: ^8.0.0`
- [ ] `lib/webview/module_webview.dart`: remove unconditional `webview_flutter_android`
      import; wrap Android-specific WebView code in `dart:io` Platform.isAndroid guard
      using conditional import pattern
- [ ] `lib/bridge/shell_bridge.dart`: platform-conditional camera (file picker on desktop)
      and graceful barcode response on desktop
- [ ] `macos/Runner/DebugProfile.entitlements`: add network client + outgoing connections
      + file read entitlements
- [ ] `macos/Runner/Release.entitlements`: same entitlements as DebugProfile
- [ ] `macos/Runner/GoogleService-Info.plist`: Firebase config for macOS
      (operator must register macOS app in Firebase Console and place file here)
- [ ] `lib/firebase_options.dart`: generated by `flutterfire configure` for
      cross-platform Firebase init on Windows (operator action)

## Operator Actions Required Before Builder Runs

1. Register **macOS app** in Firebase Console (bundle ID: `in.colligence.foundry.position`)
   → download `GoogleService-Info.plist` → place at `macos/Runner/GoogleService-Info.plist`
2. Register **Windows app** in Firebase Console
   → run `flutterfire configure` to generate `lib/firebase_options.dart`

Builder handles all code changes. Operator handles Firebase registration.

## Planner Notes

⚠ UNCLEAR-1: `module_webview.dart` Android-specific code — the import of
`webview_flutter_android` must be removed or made conditional. Validator must specify
exact pattern: either remove the import entirely (runtime check `_controller.platform is
AndroidWebViewController` still works) OR use a conditional import stub.
Recommendation: remove the import and wrap the Android block with `if (Platform.isAndroid)`.

⚠ UNCLEAR-2: `file_picker` for desktop camera — exact API call for picking an image
file on macOS/Windows. Validator must specify exact `FilePicker.platform.pickFiles()`
call with correct type filter.

⚠ UNCLEAR-3: Firebase on Windows — `flutterfire configure` generates `firebase_options.dart`.
Until operator runs this, Windows Firebase init will fail. Validator must specify how
`main.dart` should be updated to use `firebase_options.dart` if present, with a fallback
that doesn't crash if it's absent.

⚠ UNCLEAR-4: `webview_flutter_windows` minimum Windows version — requires WebView2 runtime
(built into Windows 10 1803+ and Windows 11). Validator must note this requirement.
